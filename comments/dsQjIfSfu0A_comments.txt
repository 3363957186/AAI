If you try to run it on Windows on ARM, any call to the OS/Hardware that is made in code for a x86/64bit processor will be translated to the Windows on ARM equivalent.  The App sees an emulated version of Windows x86/64.  If that emulation is not perfect, you get issues.  Also that process slows it down and you do not get native performance.
